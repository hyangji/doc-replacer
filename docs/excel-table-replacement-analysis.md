# 엑셀 대비표 기반 HWP 표 일괄 수정 — 구조 분석 및 진행 계획

> 작성일: 2026-06-02
> 상태: **분석 완료 / 개발 대기** (사용자 "진행" 지시 시 이 문서 이어서 작업)
> 분석 대상: 바탕화면 `고시문 샘플.hwp`, `고시문 샘플.xlsx`

---

## 0. 한 줄 요약

`고시문 샘플.xlsx`는 단순 찾기/바꾸기 목록이 아니라 **HWP 고시문의 표 5개 섹션을 옮겨온 "기정·변경·변경후" 개정 대비표**다.
의도는 **HWP 표의 '기정' 숫자를 엑셀의 '변경후' 값으로 바꾸는 것**.
**현재 기능으로는 불가**(포맷·숫자형식 둘 다 안 맞음). 신규 기능으로 개발하면 가능.

---

## 1. 입력 파일 구조

### 1-1. `고시문 샘플.hwp` (87,552 bytes, HWP 5.0 바이너리)
- 동두천 국가산업단지 산업단지계획 변경 고시문.
- 본문 텍스트 23,003자(한글 8,009자), **표 50개**.
- 로컬 백엔드(`HwpService`)로 텍스트/표 추출 정상.
- **숫자는 천단위 콤마 포함 표기**: `267,309`, `184,636` 등.

### 1-2. `고시문 샘플.xlsx` (26,090 bytes, 시트 5개)
각 시트 = HWP의 한 표 섹션. 시트명 번호가 HWP 항목번호와 일치.

| 시트명 | HWP 섹션 | 표 내용 | 영역 |
|---|---|---|---|
| `5. 나. 유치업종 배치계회규 (변경)` | 5.나 유치업종 배치계획 | 업종별 면적/구성비 | B2:J14 |
| `7. 사업시행지역의 토지이용현황 (변경)` | 7 토지이용현황 | 지목별·소유자별 현황 | B2:P14 |
| `8. 가. 용도지역계획 (변경)` | 8.가 용도지역계획 | 용도지역별 면적 | B1:J8 |
| `8. 나. 토지이용계획 (변경)` | 8.나 토지이용계획 | 시설용지별 면적 | D2:L12 |
| `8. 다. 주요기반시설계획 (변경)` | 8.다 주요기반시설 | 도로 등 | B1:O149 |

**공통 컬럼 패턴**: `기정(원래값) | 변경 | 변경후` 또는 `기정 | 변경 | 증감`.
- `기정` = 현재 HWP에 들어있는 값
- `변경후`(또는 `변경`) = 목표값
- `변경`/`증감` 일부는 **델타 표기**(`감) 526`, `증) 110`) — 절대값 아님.
- **엑셀 숫자는 콤마 없는 raw 값**: `184636` (HWP는 `184,636`).
- 시트마다 기정/변경후 **컬럼 위치가 다름** (예: 시트5는 기정=3·변경=4·변경후=5열, 시트8.나는 기정=4·변경=5열, 시트7은 카테고리별 기정/변경 교차).

#### 예시 (시트 `8. 나. 토지이용계획`)
```
구분        | 면적: 기정 | 변경   | 증감     | 구성비: 기정|변경|증감
합계        | 267309    | 266717 | 감) 592  | 100 | 100 | -
산업시설용지 | 184636    | 184110 | 감) 526  | 69.1| 69  | 감) 0.1
지원시설용지 | 5338      | 5338   | -        | 2   | 2   | -
공공 소계   | 77335     | 77269  | 감) 66   | 28.9| 29  | 증) 0.1
 ...
```
→ HWP에서 `184,636` → `184,110` 으로 바꾸면 되는 구조.

## 2. 의도 (사용자가 원하는 작업)

> **HWP 표 안의 '기정' 숫자들을, 엑셀의 '변경후' 값으로 일괄 교체한다.**

예: 토지이용계획 산업시설용지 면적 `184,636` → `184,110`, 합계 `267,309` → `266,717` 등.

## 3. 현재 시스템으로 안 되는 이유 (2가지 독립적 블로커)

현재 엑셀 일괄교체 기능: `backend/app/services/excel_service.py::parse_replacement_excel`
- **시트 1개**(active)에 `원본 | 수정`(또는 필드명/원본/수정) **2~3열 단순 목록**을 기대.
- 헤더 인식: `OLD_VALUE_HEADERS`(원본/찾을내용…), `NEW_VALUE_HEADERS`(수정/변경/바꿀내용…).

### 블로커 ① 포맷 불일치
이 엑셀은 시트 5개에 표 전체가 들어있음. 현재 파서에 넣으면 첫 시트의 헤더가 원본/수정과 안 맞아 fallback(`_default_col_map`)으로 0열=필드명,1열=원본,2열=수정으로 **오해석** → 엉뚱한 교체 발생.

### 블로커 ② 숫자 형식 불일치 (콤마)
- 엑셀: `267309` (raw)
- HWP: `267,309` (콤마)
- 검증 결과: `267309` → HWP 매칭 **0건**, `267,309` → 매칭 O.
- 즉 콤마 정규화 없이는 아무것도 안 바뀜.

```
267309  in HWP: False   |  267,309 in HWP: True
266717  in HWP: False   |  266,717 in HWP: True
184636  in HWP: False   |  184,636 in HWP: True
40173   in HWP: False   |  40,173  in HWP: True
```

## 4. 신규 기능으로 만들 때의 과제 (난이도순)

1. **콤마/공백 정규화** (쉬움): `184636` ↔ `184,636 ` 매칭. 엑셀값을 천단위 콤마로 포맷해서 HWP와 대조.
2. **시트별 표 구조 파싱** (중간): 시트마다 기정/변경후 컬럼 위치·헤더가 다름. 시트별 매핑 규칙 또는 헤더 자동탐지(`기정`/`변경후` 라벨 기준) 필요. 델타(`감)`/`증)`)는 제외하고 절대값(변경후) 우선 사용.
3. **숫자 중복 → 오교체 위험** (중요): `100`, `240`, `-`, `2.9` 등 값이 문서 곳곳에 반복. **전역 찾기/바꾸기는 위험.** 안전책:
   - 표·셀 단위로 위치를 특정해서 그 셀만 교체.
   - 적용 전 **Diff 미리보기**로 사람이 항목별 확인 후 적용(이미 있는 Diff/preview 흐름 활용).
4. **HWP 표 추출 정밀도** (작업량 있음): 현재 `_extract_tables_from_hwp_bytes`가 **병합셀 많은 복잡한 표에서 셀 정렬이 어긋남**(아래 증거). 셀 단위 매칭하려면 보강 필요.

### HWP 표 추출 정렬 문제 증거 (table 2, 토지이용현황)
```
['구  분','계','답','전','구','도','대','기타','기정','변경','기정','변경','기정','변경','기정']
['변경','기정','변경','기정','변경','기정','변경','필지수','240','240','154','152','7','7','32']  ← 행 경계가 밀림
['38','27','24','12','11','8','8','면적(㎡)','267,309','266,717', ...]                         ← '38'은 윗행 값
```
→ nCols 기반 평탄화가 병합셀/멀티라인셀에서 어긋남. 셀단위 매칭 전제라면 파서 개선 또는 "텍스트 근접 매칭" 대안 검토.

## 5. 제안하는 구현 방식 (안전 우선)

흐름: **대비표 엑셀 업로드 → 시트별 표 파싱 → (기정값, 변경후값) 쌍 추출 → 콤마 정규화 → HWP에서 기정값 위치 탐색 → Diff 미리보기 → 사용자 확인 → 적용**

핵심 설계 포인트:
- 교체 단위는 `(기정값, 변경후값)` 쌍. 단, **변경된 셀만**(기정≠변경후) 대상. 동일값/`-`/델타셀은 스킵.
- 매칭은 가능하면 **섹션(표) 범위 한정** 후 그 안에서 값 매칭 → 전역 오교체 방지.
- 모든 교체는 **미리보기 후 명시적 적용**(엑셀 일괄교체가 이미 preview/save 분리 구조이므로 재사용).
- 1차는 **숫자 셀**(면적/구성비)만 대상으로 좁혀 정확도 확보 → 이후 확장.
