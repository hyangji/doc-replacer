# DocReplacer 사용법 가이드

도시계획 제안서의 반복적 문서 수정을 자동화하고, 법률 문서의 정확성을 검증하는 웹 시스템입니다.

| **로컬 개발** | http://localhost:3000 |

## 1. 대시보드

접속하면 대시보드가 표시됩니다.

### 화면 구성
- **통계 카드**: 전체 문서 수, HWP/HWPX 문서 수, 법률 검색 바로가기
- **빠른 작업**: 문서 업로드 / 법률 검색 / 문서 목록 보기 버튼
- **최근 문서**: 최근 수정된 문서 5건 (파일명, 파일형식, 수정일)

### 할 수 있는 작업
- 파일을 드래그하거나 클릭하여 바로 업로드
- 최근 문서 테이블에서 **열기** 클릭 → 편집기로 이동
- 최근 문서 테이블에서 **삭제** 클릭 → 확인 후 삭제

---

## 2. 문서 업로드

### 지원 파일 형식
| 형식 | 확장자 |
|------|--------|
| 한글 문서 | `.hwp`, `.hwpx` |
| 엑셀 | `.xlsx`, `.xls` |
| 워드 | `.docx`, `.doc` |

- 최대 파일 크기: **50MB**

### 업로드 방법
1. 대시보드 또는 문서 목록 페이지의 업로드 영역에 **파일을 드래그**하거나 **클릭하여 선택**
2. 업로드가 완료되면 자동으로 **편집기 페이지**로 이동
3. HWP/HWPX 파일은 텍스트가 자동 추출되어 편집기에 표시됨

---

## 3. 문서 목록

좌측 사이드바 > **문서 목록** 메뉴에서 접근합니다.

- 업로드된 전체 문서를 테이블로 표시 (20건씩 페이지네이션)
- 파일형식별 색상 태그로 구분 (HWP=파란색, XLSX=초록색, DOCX=보라색)
- **열기**: 편집기에서 문서 열기
- **삭제**: 확인 모달 후 삭제

---

## 4. 문서 편집기

문서를 열면 3개 탭으로 구성된 편집기가 표시됩니다.

### 4-1. 편집 탭

Monaco Editor(VS Code와 동일한 편집기) 기반의 텍스트 편집 화면입니다.

**단축키:**

| 단축키 | 기능 |
|--------|------|
| `Ctrl + S` | 저장 |
| `Ctrl + H` | 찾기/바꾸기 패널 열기 |
| `Ctrl + F` | 찾기 |
| `Ctrl + Z` | 되돌리기 |
| `Ctrl + Shift + Z` | 다시하기 |

**툴바 버튼:**
- 저장 / 되돌리기 / 다시하기
- 변환 (Word/PDF) — *현재 미구현*
- 찾기/바꾸기 패널 토글
- 전체화면 토글

**하단 상태 표시줄:**
- 커서 위치 (줄, 열), 총 줄 수, 문자 수, 인코딩(UTF-8)

### 4-2. 찾기/바꾸기

편집기 우측 상단에 패널이 표시됩니다.

1. **찾기**: 검색어 입력 → 일치 항목 수 표시 (예: "3/10")
2. **옵션**: `Aa` 대소문자 구분 / `.*` 정규식 모드
3. **이동**: 이전(↑) / 다음(↓) 버튼 또는 `Enter` / `Shift+Enter`
4. **바꾸기**: 토글 아이콘 클릭 → 바꾸기 행 표시
   - **바꾸기**: 현재 일치 항목만 교체
   - **모두 바꾸기**: 전체 일치 항목 한번에 교체
5. **닫기**: `Esc` 키 또는 X 버튼

### 4-3. 엑셀 일괄 교체 탭

엑셀 파일의 데이터를 기반으로 문서 내용을 일괄 교체합니다.

**사용법:**
1. **엑셀 일괄 교체** 탭 클릭
2. 엑셀 파일(.xlsx, .xls)을 업로드
3. 매핑 미리보기 테이블에서 교체 내용 확인
4. 완료되면 자동으로 **Diff 비교** 탭으로 전환

**엑셀 파일 형식 예시:**

| 검색어 (찾을 값) | 교체어 (바꿀 값) |
|-------------------|-------------------|
| 500,000,000원 | 600,000,000원 |
| 2024년 | 2025년 |
| 일금오억원정 | 일금육억원정 |

### 4-4. Diff 비교 탭

원본과 수정본의 차이를 시각적으로 비교합니다.

**보기 모드:**
- **나란히 (Split)**: 원본과 수정본을 좌우로 나란히 표시
- **통합 (Unified)**: 하나의 뷰에서 삭제/추가를 색상으로 구분

**색상 의미:**
- 🟢 초록 배경: 추가된 내용
- 🔴 빨간 배경: 삭제된 내용

**변경사항 네비게이션:**
- **이전 변경** / **다음 변경** 버튼으로 변경 항목 간 이동
- 하단 요약 테이블에서 모든 변경사항 목록 확인

**액션:**
- **저장**: 변경사항을 최종 확정
- **모두 되돌리기**: 모든 변경사항을 취소하고 원본으로 복원
- **변경 보고서 다운로드**: 변경 내역을 파일로 내보내기

## 5. 법률 검색

좌측 사이드바 또는 상단 메뉴의 **법률 검색**에서 접근합니다.

### 화면 구성
- **좌측 (60%)**: 검색 입력 + 결과 리스트
- **우측 (40%)**: 선택한 법령의 조문 상세 내용

### 사용법

1. **검색 타입 선택**
   - **법령명**: 법령 이름으로 검색 (예: "도시계획법")
   - **조문**: 특정 조문 번호로 검색 (예: "제30조")
   - **키워드**: 내용 키워드로 검색 (예: "용도지역")

2. **검색어 입력** 후 검색 버튼 클릭 (또는 Enter)

3. **결과 리스트**에서 원하는 법령 클릭

4. **우측 패널**에서 조문 전문 확인

5. **복사** 버튼으로 조문 내용을 클립보드에 복사

> 참고: 현재 모의(Mock) 데이터로 동작합니다. 실제 국가법령정보 API 연동은 LAW_API_KEY 환경변수 설정 후 사용 가능합니다.

## 6. 주요 워크플로우

### 워크플로우 A: 금액 변경이 있는 제안서 수정

```
1. 대시보드에서 HWP/HWPX 파일 업로드
       ↓
2. 편집기에서 문서 내용 확인
       ↓
3. [엑셀 일괄 교체] 탭에서 교체 목록 엑셀 업로드
       ↓
4. [Diff 비교] 탭에서 변경사항 검토
       ↓
5. 이상 없으면 [저장], 문제 있으면 [모두 되돌리기]
```

### 워크플로우 B: 특정 단어 찾아 바꾸기

```
1. 편집기에서 문서 열기
       ↓
2. Ctrl+H로 찾기/바꾸기 패널 열기
       ↓
3. 검색어와 교체어 입력
       ↓
4. [모두 바꾸기]로 일괄 교체
       ↓
5. Ctrl+S로 저장
```

### 워크플로우 C: 법률 조문 확인 후 문서 수정

```
1. [법률 검색]에서 관련 법령 검색
       ↓
2. 조문 내용 확인 후 [복사]
       ↓
3. 편집기로 돌아가 해당 내용 붙여넣기
       ↓
4. Ctrl+S로 저장
```

## 7. 로컬 개발 환경 실행

### 프론트엔드
```bash
cd frontend
npm install
npm run dev
# http://localhost:3000 에서 접속
```

### 백엔드
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload
# http://localhost:8000 에서 접속
# http://localhost:8000/docs 에서 API 문서 확인
```

### Docker (전체 서비스)
```bash
docker-compose up -d
```
